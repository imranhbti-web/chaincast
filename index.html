Markov chains in 2 minutes
What is it?

A Markov chain assumes the next event depends only on the recent past — not the whole history. The app simply counts: "after Sunny, how many times did Rainy come next?" and turns the counts into percentages.

What does order 1 / 2 / 3 mean?

Order 1: looks at the last 1 event. "It's Cloudy now → what usually follows Cloudy?"

Order 2: looks at the last 2 events. "Sunny then Cloudy → what usually follows that pair?" Patterns like "upset always comes after two normal days" only show up at order 2+.

Order 3: last 3 events. More precise patterns, but needs much more data — with 3 states, order 3 has 27 possible contexts to learn.

How much data do I need?

Rough rule: at least 10–20× the number of contexts. With 3 states: order 1 needs ~30+ events, order 2 ~100+, order 3 ~300+. Below that, higher orders mostly memorize noise. The walk-forward test above tells you honestly.

What if my exact context was never seen before?

The app backs off: if the last-3 pattern is new, it tries the last-2 pattern, then last-1, then overall frequencies. The prediction card tells you when this happens.

File format for upload

One event per row, oldest first. CSV or Excel. If there are several columns, you'll be asked which one to use. Values are treated as text labels — "Positive", "Late", "Happy" — anything goes.
